## Project Overview
Developed a threat model for TRex Inc. to evaluate system architecture, identify potential security threats, and assess risk within a SaaS-based environment. The analysis focused on protecting sensitive data, application components, and user interactions.

## Key Focus Areas
- Analyzed system architecture including cloud components and user access points  
- Mapped data flows between users, applications, and backend systems  
- Identified threats such as unauthorized access, data exposure, and misuse of system resources  
- Evaluated risks based on likelihood and potential business impact  
- Recommended mitigation strategies to strengthen security controls  

## Threats Identified
- Unauthorized access to sensitive data  
- Weak authentication and access control gaps  
- Data exposure during transmission  
- Vulnerabilities within cloud-hosted services  

## Mitigation Strategies
- Implemented strong access controls and multi-factor authentication (MFA)  
- Applied encryption for data in transit and at rest  
- Recommended network segmentation and secure communication channels  
- Established monitoring and logging for detection and response  

## Tools and Frameworks
- Threat modeling methodology  
- Risk analysis techniques  
- CIA Triad (Confidentiality, Integrity, Availability)  
- NIST-aligned security principles  

## Outcome
Identified key risks within the TRex environment and provided actionable recommendations to reduce vulnerabilities, strengthen security posture, and improve protection of sensitive data.
